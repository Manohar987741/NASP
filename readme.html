The Northern Arizona SNP Pipeline (NASP)¶

Overview¶

NASP is a pipeline for analysis of genomic data. It is a suite of tools meant to collect and report on statistically-relevant high-confidence positions in a collection of genomes, with emphasis on variant positions, especially single nucleotide polymorphisms (SNPs). NASP expects some combination of files in FASTA, FASTQ, SAM, BAM, VCF, and SNP-matrix (TSV) format as input, and will produce output files also in those formats. As NASP is a pipeline, it expects to link a set of external tools (usually installed separately) to complete specific analysis tasks.

Getting Started¶

  1. Run the nasp command in a terminal.
  2. Select the files, folders, job scheduler, and analyzers you want to use by filling in the prompts.
  3. The jobs will run in the background and the results written to the output folder you specified in the prompts.

Repeating Analysis¶

Nasp will save file and analysis program paths and options as XML files in the output folder. These files can be used to repeat the analysis or modify it without going through the prompts.

[run-name]-config.xml:

nasp --config [run-name]-config.xml

This file stores input file and external analysis program paths necessary for nasp to repeat the analysis tasks such as searching for duplicate regions with NUCmer, aligning contigs with BWA, or calling SNPs with GATK.

matrix_dto.xml:

vcf_to_matrix --mode xml --num-threads 12 --dto-file matrix_dto.xml

This file stores resulting analysis file paths along with filter options and analysis combinations that created the files necessary for vcf_to_matrix to recreate the statistics and analysis matrices.
